Logstash需要下载logstash.conf 放在根目录
配置文件地址：https://github.com/macrozheng/mall/blob/teach/document/elk/logstash.conf

"D:\App\logstash\logstash.conf":
```conf
input {
  tcp {
    mode => "server"
    host => "0.0.0.0"
    port => 4560
    codec => json_lines
    type => "debug"
  }
  tcp {
    mode => "server"
    host => "0.0.0.0"
    port => 4561
    codec => json_lines
    type => "error"
  }
  tcp {
    mode => "server"
    host => "0.0.0.0"
    port => 4562
    codec => json_lines
    type => "business"
  }
  tcp {
    mode => "server"
    host => "0.0.0.0"
    port => 4563
    codec => json_lines
    type => "record"
  }
}
filter{
  if [type] == "record" {
    mutate {
      remove_field => "port"
      remove_field => "host"
      remove_field => "@version"
    }
    json {
      source => "message"
      remove_field => ["message"]
    }
  }
}
output {
  elasticsearch {
    hosts => "localhost:9200"
    index => "mall-%{type}-%{+YYYY.MM.dd}"
  }
}
```

配置环境变量：
```
LS_JAVA_HOME=D:\App\Java\jdk-17
```

然后运行Elasticsearch、Kibana、Logstash 下的bat文件（eg:elasticsearch.bat Kibana、Logstash 同理）

Logstash 启动命令：
```powershell
D:\App\logstash\bin>logstash -f logstash.conf
```
Logstash conf配置检查命令：
```powershell
D:\App\logstash\bin>logstash -f logstash.conf -t
```

`& "D:/App/elasticsearch/bin/elasticsearch-create-enrollment-token.bat" -s kibana`

`& "D:/App/elasticsearch/bin/elasticsearch-create-enrollment-token.bat" -s node`

3. HTTP CA 指纹（客户端不想导入证书时用指纹）

`certutil -hashfile "D:/App/elasticsearch/config/certs/http_ca.crt" SHA256`
